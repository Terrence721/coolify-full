> **Milestone (2026-07-12):** the dev environment **moved off Windows entirely, into a WSL2 Ubuntu distro's native filesystem** — the repo now lives at `/root/projects/coolify-full` inside WSL2, not under `C:\Users\...`. Reason: `docker exec coolify-vite yarn build` was taking **over 3 hours** to complete. Root-caused via `/proc/<pid>/stat`/`wchan` to `p9_client_rpc` — every file the build touched was crossing the 9P protocol bridge Docker Desktop's WSL2 backend uses to reach the Windows NTFS host filesystem, with Windows Defender re-scanning each crossing on top; the same build run natively on the Windows host (bypassing the bridge) took under 10 seconds, proving the bridge itself was the bottleneck, not the build. Moving the repo into WSL2's own ext4 filesystem means both `coolify` and `coolify-vite`'s bind mounts read/write natively with no bridge left to cross — not just for builds, for every container operation. Result: that same `yarn build` now completes in **~2 seconds**, and the full Pest suite dropped from ~150–170s to **~31s**; enabling PHP OPcache in dev the same day added a further per-request win on top. - [ ] **PHPStan baseline: down to 700 suppressed errors, from 1,306 (2026-07-16, in progress, 19 commits so far).** That many places where level-6 static analysis is effectively off; new code touching those files gets no protection against the same bug classes already baselined away. Batch 1+2 targeted the highest-leverage spots: PHPStan attributes an untyped shared-trait method's error once *per consuming class*, so a single missing return type on a trait used by all 8 database engines (or by 18 models, for `HasSafeStringAttribute`) was silently multiplying into dozens of duplicate baseline entries. Investigated converting the interface into an abstract class — confirmed it's not a simple swap: all 8 implementors (`StandalonePostgresql`, `StandaloneRedis`, etc.) already declare `extends BaseModel implements StandaloneDatabaseInstance`, and PHP doesn't support multiple inheritance, so an abstract class can't just replace the interface in place. The real fix would need the abstract class to itself `extend BaseModel` (`abstract class StandaloneDatabaseInstance extends BaseModel`), with all 8 models switching to `extends StandaloneDatabaseInstance` instead of `extends BaseModel implements StandaloneDatabaseInstance` — a genuine structural change to 8 model class declarations, not a mechanical one. Out of scope for a drive-by cleanup pass; worth a dedicated session if you want PHPStan fully clean without any baseline entries.
